## 🚀 Quick Start (Run Everything in One Command)

**Step 1 — Clone or download the project folder**
```bash
cd password_suite
```

**Step 2 — Install dependencies**
```bash
pip install passlib colorama
```

**Step 3 — Run the full pipeline**
```bash
python main.py --all
```

That's it. All 6 steps run automatically and `audit_report.txt` is generated.

## 🧩 Run Each Module Separately

### 1️⃣  Dictionary Generator
Generates a mutated wordlist from base words.
```bash
python dictionary_gen.py
```
**Output:** `wordlist.txt`

---

### 2️⃣  Hash Extractor
Creates demo hash file simulating Linux `/etc/shadow`.
```bash
python hash_extract.py
```
**Output:** `hashes.txt`

---

### 3️⃣  Brute-Force Simulator
Runs dictionary + brute-force attack against `hashes.txt`.
```bash
python bruteforce_sim.py
```
>  Requires `hashes.txt` — run `hash_extract.py` first.

**Output:** `crack_results.txt`

---

### 4️⃣  Password Strength Analyzer
Scores passwords by entropy, complexity, and patterns.
```bash
python strength_analyzer.py
```
**Output:** `strength_report.txt`

---

### 5️⃣  Report Generator
Compiles all results into a final security audit report.
```bash
python report_gen.py
```
>  Requires `crack_results.txt` and `strength_report.txt` — run previous steps first.

**Output:** `audit_report.txt`

##  Configuration (config.py)

Before running, you can customise `config.py`:

| Setting | Default | What It Controls |
|---------|---------|-----------------|
| `HASH_TYPE` | `"md5"` | Hash algorithm for attack simulation |
| `MAX_BF_LENGTH` | `6` | Max password length for brute-force |
| `CHARSET_DEFAULT` | lowercase + digits | Characters used in brute-force |
| `DEMO_ACCOUNTS` | 6 sample users | Accounts used to generate hashes.txt |
| `COMMON_SUFFIXES` | `["123","2024","!",...]` | Suffixes appended during wordlist mutation |
| `RULES` | All enabled | Toggle each mutation rule on/off |

##  Troubleshooting

| Error | Cause | Fix |
|-------|-------|-----|
| `ModuleNotFoundError: passlib` | passlib not installed | Run `pip install passlib` |
| `ModuleNotFoundError: colorama` | colorama not installed | Run `pip install colorama` |
| `[-] Hash file not found: hashes.txt` | hash_extract.py not run yet | Run `python hash_extract.py` first |
| `[-] Results file not found` | bruteforce_sim.py not run yet | Run `python bruteforce_sim.py` first |
| `[-] Wordlist not found: wordlist.txt` | dictionary_gen.py not run yet | Run `python dictionary_gen.py` first |
| `python: command not found` | Python not in PATH | Use `python3` instead of `python` |
| `bcrypt unavailable` | passlib not installed correctly | Run `pip install passlib --upgrade` |

##  Correct Run Order (Manual)

If running each file separately, always follow this order:

```
1. python config.py            (verify settings load correctly)
2. python dictionary_gen.py   (creates wordlist.txt)
3. python hash_extract.py     (creates hashes.txt)
4. python bruteforce_sim.py   (creates crack_results.txt)
5. python strength_analyzer.py (creates strength_report.txt)
6. python report_gen.py       (creates audit_report.txt)
```

Or just use: `python main.py --all`
